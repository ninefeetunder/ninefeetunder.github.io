setTimeout和setInterval详解

发表于 2019-04-05 | 分类于 JavaScript

setTimeou和setInterval是定时器。都接受一个函数和一个数字(毫秒)作为参数，不同的是setTimeout在到达时间后只执行一次函数，setInterval是每隔那段时间便执行一次函数。同时都返回一个数字ID，一般都将这个ID用一个变量保存起来，用于清除定时器。

这是两个函数普通的用法，但是这两个函数仍然需要去更深入的了解学习。

执行结果是什么，setTimeout定时了一个函数但是执行这个函数不需要等待，这个函数是不是会立即执行呢？

即使设置了函数执行等待的时间是0，匿名函数也不会立刻执行。匿名函数会等到所有的代码执行完毕后才开始执行。

我认为它们并没有立刻执行，而是在进行等待。到达时间后就执行里面的匿名函数。这种理解完全时错误的。

其实setTimeout和setInterval是立即执行的，它们会先执行函数上的一些处理（并不是执行函数–例如给函数绑定this什么的），然后把函数送入一种名为队列的数据结构中，而进入队列的顺序由第二个参数来决定的，时间越小，越先进入队列。等到当前所有的代码都执行完毕后，JS引擎才会来到这个队列中开始执行这个队列中的函数

这就能解释上面的情况了，执行到setTimeout时函数被送进一个队列，接下来继续执行console.log，当前代码执行完毕才去执行队列中的任务。
