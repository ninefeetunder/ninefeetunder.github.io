初识Redux

发表于 2019-03-23 | 分类于 Redux

最近简单的学习下redux，对redux的理解还不够深刻，先记下再说。

  1. store就是一个存放数据的仓库，将所有组件的数据集中在store中。同时store中的数据也可以被各个组件访问到。

  2. store是唯一的。只有store自己才能改修改store中的内容。

  3. 当存在很多组件时并且每个组件都有自己的数据时，可以为每个组件创建一个reducer，把index作为出口文件，最后在总的reducer中通过调用combineReducers把所有的reducer组合成一个总的reducer，最后在利用该reducer创建store，这样store中就拥有了各个组件对应的数据。

  4. （小）reducer就是一个接收state和action的，进行逻辑处理，返回更新后数据，的纯函数。

  5. reducer函数只能接收state,不能也不允许直接修改state中的值（纯函数）。它需要把修改路径的对象都复制一遍产生一个新的对象，根据action去修改对应的值，然后返回这个新的对象

  6. reducer中要定义这个组件需要使用的数据(defaultState)，store中关于该组件的数据就是参照这个值创建的，所以这个defaultState是reducer函数state参数的初始值。

  7. reducer返回数据给store，store根据这些数据比对自己内部中存储的数据 ，一旦发现数据改变了，就修改自己内部的数据（对应第2条）。

  8. dispatch把action发送给store，store这时候并不会去自己去修改数据。所以store又把action发送给reducer（store发送的action可以被每一个reducer接收到），让reducer进行处理后，store根据reducer返回的数据更新自己内部的数据（对应第7条）。

  9. 还是jio的react-redux更方便使用。使用react-redux的关键: Provider, connect。
