JavaScript中this对象

发表于 2018-08-12 | 分类于 JavaScript

学习JavaScript函数时这一章节时，this对象是有点坑的地方。其他人我不知道，在我看了教程之后感觉还是懵的。简单点的还可以分析的出来，复杂点的就只能懵逼了。为了搞清楚this，上网找了很多资料，这里 是说的很清楚的一篇。

在JavaScript中this对象只会在函数中需要被确定，在函数中存在4中不同的情况。如果是在全局作用域下，则this就是全局对象，也就是window，其他情况如下：

  • 在单独的函数中出现

    这里的函数是作为单独的声明出现，而不是作为属性被调用

这种情况下，foo函数体中的this永远为Global对象，也就是window对象。

  • 在作为属性的函数中出现

    这里的函数是作为一个对象的属性。当对象调用这个属性，也就是执行这个函数。

函数foo为对象o的属性，这时函数体内的this对象永远为调用这个函数的对象，这里就是对象o。

  • 被new用来构造对象的函数中出现

    使用new来构造出一个对象。

这种模式下，函数体中的this永远为new foo()返回的对象。

  • 在使用了apply或者call内置函数的函数中出现

    使用了apply和call内置函数。

    foo函数使用了内置的apply和call函数，apply和call的第一个参数就是foo函数中的this。

this是在函数调用时就已经完全确认对象了，根据以上四种方式就可以很快的确定函数中的this指的是哪个对象。
