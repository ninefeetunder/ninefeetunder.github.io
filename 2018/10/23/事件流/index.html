事件流

发表于 2018-10-23 | 分类于 JavaScript

事件

事件就是用户或浏览器自身执行的某种动作。例如click,load和mouseover，它们都是事件的名字（事件名字）。JavaScript与HTML的交互使用过事件实现的。

事件流

事件流描述的就是从页面中接收事件的顺序。
当你点击一个元素时，你也点击了它的容器元素，也点击了整个文档元素。于是事件便有了一个传递的过程。

事件会按照给定的顺序一层一层的传递下去并被执行。
事件流有两种模型。

事件冒泡模型

事件冒泡，即事件开始时由最具体的元素（文档中嵌套层次最深的那个节点）接收，然后逐级的向上传播到较为不具体的结点。
以之前HTML页面为例，当点击”click me!”的div元素时，那么这个click事件会按照如下顺序传播：

div -> body -> html -> document
IE9， Firefox， Chrome，Safari会将事件一直冒泡到window对象。

事件捕捉模型

事件捕获是从不太具体的节点最先接收事件，而最具体的节点则最后接收到事件。当点击上述div元素时，click事件会按照如下顺序传递：
document -> html -> body -> div
尽管”DOM2级事件”规范要求事件应该从document对象开始传播，但是这些浏览器都是从window对象开始事件捕获。

上述两种模型，推荐使用事件冒泡，在有特殊的情况下在使用事件捕获~~~

DOM事件流

DOM事件流，我的理解就是在浏览器中触发事件后，浏览器做出响应的过程。
DOM事件流包括三个阶段： 1. 事件捕获阶段，2. 处于目标阶段 3. 事件冒泡阶段。

  1. 事件捕获阶段：document接收事件并逐级向下传播。
  2. 处于目标阶段：目标节点接收事件，执行注册在该节点上的事件处理程序。
  3. 事件冒泡阶段： 事件又被传播会文档。
    只要在浏览器中触发事件，浏览器就会执行上面的流程，完成对事件的响应。其中”处于目标”阶段被看成是事件冒泡阶段的一部分。

事件处理程序

为了响应(处理)某个事件的函数就叫做事件处理程序（或事件侦听器），事件处理程序的名字是以”on”开头，因此click事件的事件处理程序就是”onclick”,load事件的事件处理程序就是”onload”。
你若触发，我便执行———事件发生，调用事件的事件处理程序。

DOM0级事件处理程序

DOM0级事件处理程序需要，事件处理程序名字和事件处理程序函数。

使用DOM0级方法指定的事件处理程序被认为是元素的方法，该函数是在元素的作用域中运行。

DOM2级事件处理程序

主要使用addEventListener和removeEventListener这两个方法为元素指定和删除事件处理程序，它们都接受3个参数：

  1. 要处理事件的名称 ( 注意是名称 )
  2. 作为事件处理程序的函数。
  3. 一个布尔值（true表示在捕获阶段调用事件处理程序，false表示在冒泡阶段调用事件处理函数。多为false与DOM流对应）
